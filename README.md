# Graded Project on Orchestration and Scaling
Graded Project on Orchestration and Scaling in Jenkins

## Project Workflow

### Step 1: Version Control with Git
1. Fork the main repository into your GitHub account.
  1. Open the repo link - `https://github.com/UnpredictablePrashant/SampleMERNwithMicroservices.git`
  2. Click on `fork`
  3. The main repo is now forked to the Git Hub account - <img width="1553" height="566" alt="image" src="https://github.com/user-attachments/assets/e7297417-78c3-4721-9cbf-fdc513d2d3be" />
2. Clone the fork on the local machine - `git clone https://github.com/suhailm67-sys/Project-on-Orchestration-and-Scaling-in-Jenkins.git` - <img width="1863" height="206" alt="image" src="https://github.com/user-attachments/assets/e3b782bf-b114-4cfc-8a02-df06d6b9e0d8" />
3. Verify remote - `git remote -v`
4. Add Upstream Repository - `git remote add upstream https://github.com/UnpredictablePrashant/SampleMERNwithMicroservices.git`
5. Sync with Upstream
  1. Whenever the original repository changes, fetch latest - `git fetch upstream`
  2. Switch - `git checkout main`
  3. Merge - `git merge upstream/main`
  4. Push to your GitHub - `git push origin main`
